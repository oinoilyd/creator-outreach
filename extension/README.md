# Creator Outreach browser extension

Adds a small pill to YouTube channel pages **and** video pages:
one click adds that creator to your Creator Outreach board (or shows
"✓ In your outreach" if they're already there, matched by real channel
id so it dedupes against search-added leads).

Plain Manifest V3 JavaScript — no build step. The same code runs in
Chrome, Edge, Brave, and Arc, and converts to Safari (below).

## How it works

- `content.js` watches YouTube's SPA navigation and classifies the page
  (channel root or watch page).
- `background.js` re-fetches that page's HTML (cookieless) and extracts
  the channel identity from YouTube's embedded JSON — deliberately not
  DOM scraping, because the inline data goes stale on SPA navigation
  and DOM classes churn. Regexes live in `parseChannelHtml()` and are
  unit-testable from node (see Testing).
- API calls hit `https://creatoroutreach.net/api/v1/leads` with the
  platform API key (`co_live_…`) the user pastes into the popup
  (created in-app: hamburger → Integrations → Platform API keys).
  The key is stored in `chrome.storage.sync`, never sent anywhere but
  creatoroutreach.net.

## Install for development (Chrome / Edge / Brave / Arc)

1. Open `chrome://extensions`
2. Toggle **Developer mode** (top right)
3. **Load unpacked** → select this `extension/` folder
4. Click the extension icon → paste an API key → Save → Test connection
5. Visit any YouTube channel — the pill appears bottom-right

After editing files, hit the ↻ reload button on the extension card.

## Testing the extractor

```bash
node -e "
const { parseChannelHtml } = require('./extension/background.js')
// fetch any channel/watch page HTML and pass it with 'channel' | 'watch'
"
```

(The service-worker/chrome globals are guarded, so the file requires
cleanly in node.)

## Safari

Safari runs WebExtensions wrapped in a Mac app. With Xcode installed:

```bash
xcrun safari-web-extension-converter extension/ --project-location extension-safari --app-name "Creator Outreach"
```

Then open the generated Xcode project, run it once, and enable the
extension in Safari → Settings → Extensions (allow unsigned extensions
in the Develop menu during development). App Store distribution needs
an Apple Developer account ($99/yr).
